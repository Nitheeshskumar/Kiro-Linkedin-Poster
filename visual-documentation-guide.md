## Recording Guidelines

### Technical Specifications
- **Resolution**: 1920x1080 minimum
- **Frame Rate**: 30 FPS
- **Format**: MP4 (H.264)
- **Audio**: Clear narration with background music
- **Duration**: 1-3 minutes per recording

### Content Guidelines
- **Focus on Value**: Show how Kiro accelerates development
- **Highlight Quality**: Demonstrate code quality and best practices
- **Show Real Usage**: Actual application functionality, not mockups
- **Professional Presentation**: Clean, well-lit recordings with clear audio

### Editing Notes
- **Smooth Transitions**: Use fade-in/fade-out between sections
- **Callout Annotations**: Highlight important code sections and UI elements
- **Progress Indicators**: Show time savings and efficiency gains
- **Professional Branding**: Include Kiro and AWS logos appropriately

## Screenshot Capture Instructions

### Development Process Shots
1. **Clean Workspace**: Ensure Kiro interface is clean and professional
2. **Highlight Key Features**: Use arrows or highlights to draw attention
3. **Show Real Code**: Actual generated code, not placeholder text
4. **Consistent Styling**: Use same theme/colors throughout

### Application Interface Shots
1. **High Resolution**: Capture at 2x resolution for crisp display
2. **Real Data**: Use actual generated content, not lorem ipsum
3. **Multiple States**: Show loading, success, and error states
4. **Cross-Browser**: Capture in Chrome, Firefox, and Safari

### Code Quality Shots
1. **Syntax Highlighting**: Ensure proper code formatting
2. **Full Context**: Show enough code to understand functionality
3. **Comments Visible**: Include Kiro-generated documentation
4. **File Structure**: Show organized project layout

This visual documentation will provide compelling evidence of Kiro's development acceleration capabilities while showcasing the professional quality of the generated application.
